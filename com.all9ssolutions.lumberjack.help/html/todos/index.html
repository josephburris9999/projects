Custom Statements

Lumberjack fulfills its design in generating logging statements and alleviating a large part of the logging burden from programmers. However, the plug-in is still early in its lifecycle. Currently, Lumberjack is dependent on template statements which reside in the archive. A planned enhancement is to allow Lumberjack to depend on either the existing template statements, or project specific template statements.

Lost Comments

Comments after return statements are over-written when Lumberjack is required to create a variable for return. Because of the Eclipse document format, each statement is treated as a node. When Lumberjack sees that a literal value or a method call is being returned from a method, it creates a variable with the value of the return statement. This is done so the return value may be logged (if the option is selected) and an exiting statement may be inserted at the end of the method, but prior to the return statement. Within Eclipse comments are associated with each node rather than their position in the document, so when Lumberjack creates a node the comments are unable to be carried over to the new return statement.

Multiple Classes

While Lumberjack only supports logging the Java class in the current editor, it may be convenient to log more classes at once. A planned enhancement is to allow running Lumberjack for a selection of multiple classes, a package of classes, or all packages for a project, at one time.
